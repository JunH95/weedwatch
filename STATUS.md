## ✅ Stage 0 — 스모크 (완료)

**질문**: 사람이 안 보고 있어도 시뮬을 돌려서 진짜 GPU 사진을 받아올 수 있는가?
이게 안 되면 "Claude가 스스로 검증한다"는 프로젝트의 대전제가 무너진다.

**답: 된다.** `make smoke` 통과.

```
사진 장수   : 121
평균/표준편차: 142.00 / 36.12
GL_VENDOR   : NVIDIA Corporation
GL_RENDERER : NVIDIA GeForce RTX 4060 Laptop GPU/PCIe/SSE2
```

가는 길에 함정 3개를 밟았고 전부 `CLAUDE.md`에 기록했다:

1. **EGL이 인텔 내장 그래픽을 잡고 있었다.** RTX 4060이 EGL 장치 목록에 아예 없었다.
   NVIDIA ICD를 고정해서 해결. → 그래서 모든 시뮬 테스트는 게이트가 2개다
   (사진이 나왔나 **AND** NVIDIA가 그렸나). 사진만 보면 거짓 통과한다.
2. **카메라는 구독자가 있을 때만 렌더링한다.** 구독자 0 → 사진 0장, 에러 메시지 없음.
   `tools/run_headless.sh`가 구독자를 붙여준다.
3. **`pkill -f 'ign gazebo'`는 자기 자신을 죽인다.** 자기 명령줄에도 그 문자열이 있어서.
   `'[i]gn gazebo'`로 해결.

## ▶️ 다음 — Stage 1: 정원과 오라클

성공 기준("잡초 위에 정확히 선다")을 **측정할 수 있게** 만드는 단계.
측정 못 하면 검증도 없고, 검증이 없으면 이 프로젝트는 성립하지 않는다.

      — GUI 클릭이 필요한데 우리에겐 GUI가 없다. 안 켜면 조용히 CPU로 렌더링한다.
- [ ] **두둑/고랑 지형 직접 제작** ← 이번 단계 최대 위험.
      CropCraft에는 지형 개념이 아예 없다(바닥이 평면). 포탈형 로봇 설계 전체가
      두둑 높이 25cm에 걸려 있다. 반나절 실험으로 먼저 확인할 것.
- [ ] CropCraft 설치 → 주말농장 YAML → Gazebo로 내보내기
- [ ] **머티리얼 변환 패스** — 안 하면 정원 전체가 새까맣게 렌더링된다
      (CropCraft가 내보내는 Ogre1 머티리얼을 gz-sim이 무시하고 검정으로 덮어씀)
- [ ] **채점 대상 잡초를 `weeds:`가 아니라 `beds:` 블록에 선언**
      → `description.json`에 정확한 x,y,z 좌표가 나온다 = 오라클 확보

**Stage 1 완료 조건**: 같은 시드로 다시 만들면 똑같은 `description.json`이 나오고,
